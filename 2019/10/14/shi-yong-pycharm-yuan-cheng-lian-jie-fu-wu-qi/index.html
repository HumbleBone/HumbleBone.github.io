使用Pycharm远程连接服务器

困惑👾：

我们在调试机器学习有关的python工程时，总会遇到这样的问题。就是，如何实现能够同步本地和服务器端的工程项目。正常的操作是👀，我们打开终端（比如MobaXterm）…… 然后，将本地编辑好的文件上传到服务器，这时候在终端再运行程序，如此往复……

又或者是在服务器端直接调试程序，然后下次要在本地编辑的话，又要重新从服务器上下载代码，极其不方便🙁。

那么，有没有更好的方法，让这种操作一体化，简去这种繁琐的操作🤔，答案是有😏。

利用 PyCharm ，我们可以实现：

  • 在本地利用远程服务器的python编译器调试代码
  • 将本地修改好的代码同步到服务器（可以自动同步）
  • 将服务器上的项目直接下载下来

具体操作

下载PyCharm

PyCharm 分为专业版和社区版，我们要使用的远程功能只有专业版才有提供，如果你是在校的学生，你可以直接在PyCharm的官网上申请。如果你直接申请过 Github 的学生优惠包🛍️，你可以直接验证邮箱，之后注册一个账号（这个账号后面会用到），具体申请过程这里就不细说了。

配置Deployment

1、我们打开pycharm工程界面后，选择 tools –> Deployment –> Configuration

2、选择 + 新建一个 sftp 类型的部署配置，服务器名字自己随便起一个（易于区分就行）。然后，我们就可以设置服务器的地址和端口号（默认22，如果有修改填自己的端口号），用户名密码。

如果连接成功的话，会出现如下所示的界面。

3、设置mappings，用于项目的上传和下载。对于不同的项目，你可以设置多个映射。

4、 另外打开 Tools -> Deployment -> Options，将 Create Empty directories 打上勾，要是指定的文件夹不存在，会自动创建。

选择远程解释器

1、打开 file –> settings ， 然后在右边，点击那个小齿轮进行设置。 选择项目的解释器，在项目的解释器位置选择新增。

2、添加ssh解释器

3、上述步骤结束后，就可以在本地调试代码，并部署到服务器上。
